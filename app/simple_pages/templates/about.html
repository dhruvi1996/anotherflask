PROJECT OUTLINE

Another flask project : it is the root of the project which contains all the files, images necessary for the application. Below explained all the files

•.github : this folder contains the Workflow files of Github –>Actions

 prod.yml : this file contains all production deployment Workflow in Github->Actions.

 dev.yml : this file contains all developmental deployment Workflow in Github->Actions.

• .pytest_cache : this folder holds all the records running in the project during the process of pytest

• app : this is the application folder with few files.

context_processors : to inject new variables automatically into the context of a template, so context processors exist in Flask.

simple_pages : this folder has templates: it consists of individual page .html pages, and Flask Blueprint controller (init_py)

static : the static folder contains assets used by the templates, including CSS files, JavaScript files, and images.

templates : this folder contains base.html file which has the DOCTYPE, html tag, head tag, meta tag, title tag, and body tag, where markup from other templates will be dynamically inserted at runtime.

init__.py : The init__.py file provides the interface to the app module.

run.py : This is python file which allows to serve application in production.

• calculator : not used file for this application

• tests : in this folder testing helps ensure that your app will work as expected for your end users. So it has files which allow framework for running in the application. It has simple_pages_test.py, context_process_test.py, conftest.py , __init_py.

• .coveragerc : this file is in the same directory as coverage.py it is its controller.

• .dockerignore : file allows you to exclude files from the context like a . gitignore file , allowing you to exclude files from your git repository. It helps to make build faster and lighter by excluding from the context big files or repository that are not used in the build.

• .gitignore : this file tells Git which files to ignore when committing your project to the GitHub repository.

• .pylintrc : Pylint searches up the hierarchy of Python packages until it finds a pylintrc file. This allows to specify coding standards on a module-by-module basis.

• docker-compose.yml : Using Docker Compose, we can work on different components of the project on different Docker Containers and combine them to create a single application. It allows us to build multi-container Docker applications.

• Dockerfile : this file is a text document that contains all the commands a user could call on the command line to assemble an image. We can create an image which runs in the container in the program application.

• heroku.yml : this file is a manifest you can use to define our Heroku app. It allows to build docker images on Heroku.

• pytest.ini : this file is used to hold pytest configurations ; what functions to be executed or ignore.

• Readme.md : this is a Read Me file in which instructions, links , text, images are there can be used to set our project's description.

• Requirements.txt : this files contains all the necessary packages needed to install in order to run the application.

• setup.py : this file allows us to easily install Python packages for the running of application.
